Format: Start with `#!/bin/bash` then `set -euo pipefail`.

Key structure:

2. **Bootstrap flag (BOOT-07 idempotency)**: Check for /var/lib/lightrag/.bootstrapped. If exists, skip full bootstrap and just ensure services are running.
3. **Swap file (BOOT-01)**:
   - Check if already enabled: `swapon --show | grep /swapfile` and skip if present
   - Commands: `fallocate -l 2G /swapfile || dd if=/dev/zero of=/swapfile bs=1M count=2048`, `chmod 600 /swapfile`, `mkswap /swapfile`, `swapon /swapfile`, add to /etc/fstab
4. **Docker install (BOOT-02)**:
   - `apt-get update -y`
   - `apt-get install -y docker.io docker-compose-v2 awscli curl`
   - `systemctl start docker && systemctl enable docker`
   - `usermod -aG docker ubuntu` (allow ubuntu user to access Docker socket)
5. **Create app directories**: /opt/lightrag, /var/lib/lightrag, /var/lock
6. **Clone LightRAG repo (BOOT-03)**:
   - Only if /opt/lightrag/.git does not exist (idempotent per BOOT-07)
   - `cd /opt && git clone https://github.com/HKUDS/LightRAG.git lightrag`
   - `cd /opt/lightrag`
7. **Restore rag_storage from S3 (BOOT-04)**:
   - Run `aws s3 sync s3://${s3_bucket_name}/rag_storage/ /opt/lightrag/data/rag_storage/ || true`
   - If S3 is empty (first boot), this gracefully does nothing
8. **Load secrets from SSM (BOOT-05)**:
   - Function to retry SSM get-parameter up to 3 times with 5-second backoff
   - Fetch /lightrag/ANTHROPIC_API_KEY, /lightrag/OPENAI_API_KEY, /lightrag/LIGHTRAG_API_KEY

   - Write to /opt/lightrag/.env with template values:

     ```
     ANTHROPIC_API_KEY=<from SSM>
     OPENAI_API_KEY=<from SSM>
     LIGHTRAG_API_KEY=<from SSM>
     MODEL=claude-sonnet-3-7-20250619
     MODEL_LIST= claude-haiku-3-5-20250514
     EMBEDDING_MODEL=text-embedding-3-large
     EMBEDDING_DIM=3072
     ```
   - NOTE: Embedding model config (EMBEDDING_MODEL=text-embedding-3-large, EMBEDDING_DIM=3072) comes from the LightRAG env.example defaults. Do NOT hardcode MODEL values -- use SSM-loaded values as the source of truth.

   - Copy the local docker-compose.override.yml to /opt/lightrag/docker-compose.override.yml
   - Ensure memory limit (750m) and health check are in the override

    - `cd /opt/lightrag && docker compose up -d`
    - Wait for health: `docker compose ps` polling up to 60 seconds
12. **Bootstrap complete**:
    - Create /var/lib/lightrag/.bootstrapped flag file
    - Log completion with timestamp

### scripts/sync-rag-storage.sh
S3 sync script with flock locking. Format: `#!/bin/bash` + `set -euo pipefail`.
- Lock file: /var/lock/lightrag-s3-sync.lock
- Source: /opt/lightrag/data/rag_storage/ (note: upstream LightRAG uses /app/data/rag_storage in the container, bind-mounted from ./data/rag_storage on the host -- so on the EC2 host, the path is /opt/lightrag/data/rag_storage/)
