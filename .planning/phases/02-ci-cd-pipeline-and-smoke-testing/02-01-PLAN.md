Create `.github/workflows/deploy.yml` with the following structure:

**Trigger:** `push.tags` with pattern `v*` (e.g., `v0.1.0`, `v1.0.0`).

**Concurrency:** `cancel-in-progress: true` (same repository group).

**Environment variables:**
- `EC2_HOST: 54.253.108.90`
- `EC2_USER: ubuntu`
- `LIGHTRAG_IMAGE: ghcr.io/hkuds/lightrag`

**Steps:**

1. `actions/checkout@v4` -- full checkout so git pull works on the instance

2. `aws-actions/configure-aws-credentials@v4` with OIDC:
   - Use `actions/github-script@v7` to fetch the OIDC token: `const token = await core.getIDToken('sts.amazonaws.com')`
   - Set env: `AWS_ROLE_TO_ASSUME: arn:aws:iam::$(aws sts get-caller-identity --query Account --output text):role/mythicc123`
   - Set env: `AWS_WEB_IDENTITY_TOKEN_FILE: ${{ github.action_path }}/oidc_token.txt`
   - Set env: `AWS_DEFAULT_REGION: ap-southeast-2`
   - audience: `sts.amazonaws.com`
   - Note: The mythicc123 role must already exist; do NOT create it. The role ARN is constructed dynamically using the AWS account number from STS.

3. `appleboy/ssh-action@v1.2.5` with:
   - host: `${{ env.EC2_HOST }}`
   - username: `${{ env.EC2_USER }}`
   - key: `${{ secrets.EC2_SSH_KEY }}`
   - timeout: `10m`
   - script: |
     ```
     set -euo pipefail
     DEPLOY_DIR="/opt/lightrag"
     echo "=== Starting deployment at $(date -u '+%Y-%m-%dT%H:%M:%SZ') ==="

     # --- Digest pinning (CICD-04) ---
     # Fetch the SHA256 digest of the :latest tag so deployments are reproducible.
     # Using :latest is insecure -- the tag can be overwritten with a different image.
     # Pinning to a digest locks the image to an immutable content address.
     echo "[Digest] Fetching ghcr.io/hkuds/lightrag:latest digest..."
     DIGEST=$(docker manifest inspect ghcr.io/hkuds/lightrag:latest --format '{{.Digest}}')
     if [ -z "$DIGEST" ]; then
       echo "ERROR: Could not fetch image digest"
       exit 1
     fi
     IMAGE_WITH_DIGEST="ghcr.io/hkuds/lightrag@${DIGEST}"
     echo "[Digest] Using image: ${IMAGE_WITH_DIGEST}"

     # Navigate to deploy directory
     cd "$DEPLOY_DIR"

     # Pull latest changes from git (CICD-03)
     echo "[Git] Pulling latest..."
     git fetch --tags
     git pull origin master

     # --- Update docker-compose.override.yml with digest-pinned image ---
     # Create or update docker-compose.override.yml to pin the image digest
     cat > docker-compose.override.yml << 'COMPOSE_EOF'
     services:
       lightrag:
         image: ${IMAGE_WITH_DIGEST}
         restart: unless-stopped
         stop_grace_period: 30s
         deploy:
           resources:
             limits:
               memory: 750m
         healthcheck:
           test: ["CMD", "curl", "-f", "http://localhost:9621/health"]
           interval: 10s
           timeout: 5s
           retries: 5
           start_period: 60s
         environment:
           HOST: "0.0.0.0"
           PORT: "9621"
     COMPOSE_EOF

     # Override the IMAGE_WITH_DIGEST env var in .env so docker-compose uses it
     # The .env file is loaded by docker compose; we set the override
     {
       grep -v '^LIGHTRAG_IMAGE=' .env 2>/dev/null || true
       echo "LIGHTRAG_IMAGE=${IMAGE_WITH_DIGEST}"
     } > .env.tmp && mv .env.tmp .env

     # Pull the new image (CICD-03)
     echo "[Docker] Pulling ${IMAGE_WITH_DIGEST}..."
     docker compose pull

     # Restart containers (CICD-03)
     echo "[Docker] Restarting containers..."
     docker compose up -d

     # Wait for startup (CICD-03)
     echo "[Health] Waiting for LightRAG to be ready..."
     sleep 15

     # Health check (CICD-03)
     echo "[Health] Checking /health endpoint..."
     HTTP_CODE=$(curl -s -o /dev/null -w "%{http_code}" -f http://localhost:9621/health)
     if [ "$HTTP_CODE" != "200" ]; then
       echo "ERROR: Health check failed with HTTP $HTTP_CODE"
       exit 1
     fi
     echo "[Health] PASS -- HTTP 200"

     echo "=== Deployment complete at $(date -u '+%Y-%m-%dT%H:%M:%SZ') ==="
     ```

- `HEADERS = {"x-api-key": API_KEY, "Content-Type": "application/json"}`

**Test functions (all use pytest, all include the API key header):**

1. `test_health_check()` -- TEST-05
   - GET /health
   - Assert response status == 200

2. `test_ingest_document()` -- TEST-01, TEST-05
   - POST /documents/insert with JSON body: `{"text": "The Sydney Harbour Bridge is an iconic Australian landmark connecting the city of Sydney to the North Shore. It was opened in 1932 and spans 504 metres across the harbour. The bridge is made of steel and features a distinctive arch design."}`
   - Capture the returned task_id (or response id)
   - Return the task_id for use by the next test
   - Assert response status == 200

3. `test_ingestion_complete(task_id)` -- TEST-02, TEST-05
   - Poll GET /documents/status?task_id={task_id} every 5 seconds
   - Maximum 60 seconds total polling time
   - Break when status indicates completion (check for "completed", "done", "success", or absence of "pending" / "processing")
   - Assert the ingestion completes within 60s
   - If timeout, fail with descriptive message

4. `test_query_hybrid()` -- TEST-03, TEST-05
   - POST /query with JSON body: `{"query": "Tell me about the Sydney Harbour Bridge", "mode": "hybrid"}`
   - Assert response status == 200
   - Assert response contains a 'result' or 'answer' field with non-empty content
   - Store the response JSON for the next test

5. `test_entity_in_response()` -- TEST-04, TEST-05
   - Uses the response from test_query_hybrid (via pytest fixture or module-level variable)
   - Assert that the response contains at least one entity or relation keyword:
     - Check for presence of "Sydney", "Harbour Bridge", "Sydney Harbour Bridge", "1932", "landmark", "arch", or similar
     - OR check that the response result length > 0
   - If the response is structured (e.g., has 'entities' or 'relations' keys), assert those arrays are non-empty

**Pytest fixtures:**
- `api_key` fixture: yields the LIGHTRAG_API_KEY from env
- `session_headers` fixture: returns headers dict with API key
